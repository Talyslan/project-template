## Main User Flows

### Core Flow

1. User accesses the application
2. User creates or logs into an account
3. User views existing items (tasks, records, etc.)
4. User creates, updates, or moves items
5. User tracks progress and completion

### Management Flow

1. User edits or deletes existing items
2. User changes status or visualization mode
3. User updates personal preferences

## Onboarding Flow

1. First access to the application
2. Account creation (sign up)
3. Initial configuration or tutorial (optional)
4. First interaction with the core feature

The onboarding process should be short, clear, and focused on delivering immediate value.
