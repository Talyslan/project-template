## ğŸ·ï¸ Title & naming guidelines

To keep issues and pull requests organized, please use the following prefixes in titles:

### Issue & PR prefixes

- **feat:** a new feature or improvement  
  _Example: `feat: add database documentation template`_

- **fix:** a bug fix  
  _Example: `fix: broken link in README template`_

- **docs:** documentation-only changes  
  _Example: `docs: improve contributing guidelines`_

- **style:** formatting, whitespace, or wording changes  
  _Example: `style: improve section spacing in README`_

- **chore:** maintenance or tooling changes  
  _Example: `chore: update repository structure`_

This helps maintainers quickly understand the purpose of each change.
